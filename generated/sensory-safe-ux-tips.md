# Sensory-Safe UX Tips for AAC

**Caly guidance · CAL-AAC-UX-v0 aligned**

Sensory-safe design reduces cognitive load and supports assent. These tips apply to apps, paper boards, and classroom displays.

## Visual design

1. **Limit simultaneous motion.** No auto-rotating carousels, pulsing buttons, or surprise animations on AAC screens.
2. **Prefer muted palettes.** Dark blue-gray backgrounds with soft accent colors reduce glare. Avoid pure white full-screen flashes.
3. **Large touch targets.** Toddler tiles: minimum **88×88 px** with **12 px** gaps. Teens and adults may prefer smaller dense grids—offer choice.
4. **One concept per tile.** Label with a single word or short phrase. Avoid sentences on toddler boards.
5. **Consistent layout.** Keep **help**, **stop**, and **all done** in the same locations across boards when possible—muscle memory matters.

## Audio and environment

1. **Low, calm caregiver voice** when reading symbols aloud—not loud "teaching voice."
2. **Reduce background noise** during AAC practice (TV off, notifications silenced).
3. **No sound effects required** for symbol taps unless the user explicitly prefers them.

## Interaction patterns

1. **Wait time:** 5–10 seconds after modeling before prompting again.
2. **Honor stop immediately.** Never animate or flash when **stop** is tapped.
3. **No timers that punish.** Optional gentle timers for transitions only with assent.
4. **Offline tolerance:** Core boards should remain usable without network; cloud features are additive.

## Age-specific notes

| Band | UX priority |
|------|-------------|
| Toddler | Big tiles, minimal text, caregiver co-reading |
| Child | Categories with clear borders, break symbol always visible |
| Teen | Privacy, no childish clipart unless chosen, text labels OK |
| Adult | Dense vocabulary, accommodation symbols, export/share options |

## What to avoid

- Flashing red "error" states when the user selects "wrong" symbols
- Requiring eye contact to confirm a message
- Locking navigation until a task is completed
- Gamification streaks that punish missed days

## When sensory overload happens

Offer **break**, dim the screen if the app allows, and return later. Overload is communication—not non-compliance.
